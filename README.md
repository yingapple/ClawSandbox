# ClawSandbox

一个最简洁易用的 OpenClaw 虚拟化部署沙盒工具。让任何人都能轻松部署和管理多个 OpenClaw 实例。

## 背景

LLM AI 的应用层发展分三个阶段：
1. **ChatBot** — 让人人获取知识
2. **Agent** — 让人人成为专业的人
3. **OpenClaw** — 让人人成为管理者

OpenClaw 是一个自托管的个人 AI 助手，能连接 WhatsApp、Telegram、Slack 等 20+ 消息平台。但目前部署单个 OpenClaw 实例仍有一定门槛，更不用说同时管理多个实例。

ClawSandbox 的目标：**不应该让 OpenClaw 单体的获取成为卡点，而应该能轻松部署一个 OpenClaw 军团。**

## 核心问题

1. **部署门槛高** — OpenClaw 需要 Node.js ≥22、Chrome/Chromium、多个平台的认证配置，对非技术用户不友好
2. **多实例管理难** — 想同时运行多个 OpenClaw（比如不同用途、不同账号）没有统一的管理方式
3. **实例间隔离** — 多个实例的数据、会话、配置需要互相隔离，避免冲突

## ClawSandbox 要做什么

### 一键创建沙盒实例
- 用户执行一条命令即可创建一个完整的、隔离的 OpenClaw 运行环境
- 每个沙盒包含独立的 OpenClaw 进程、Gateway、配置和数据

### 多实例生命周期管理
- 创建、启动、停止、销毁沙盒实例
- 查看所有实例的运行状态
- 实例间资源隔离

### 降低使用门槛
- 不需要用户手动安装 Node.js、Chromium 等依赖
- 沙盒内自包含所有运行时依赖
- 面向所有人，包括非技术用户
